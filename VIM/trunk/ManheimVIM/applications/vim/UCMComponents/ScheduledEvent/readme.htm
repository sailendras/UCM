ScheduledEvent

This component demonstrates how to hook into the content server scheduling system. This system is meant for processes that take some time to complete, so they never get executed more than once every five minutes. Typically, they are rarely executed more than once per day.

The Java code leverages two hooks, 'checkScheduledEvents' and 'scheduledSystemEvents'. The former will be executed about once every five minutes. The latter is used along with special IdocScript resources to conditionally execute at most once per hour. In this way, you can configure an event to fire every hour, or every night at midnight.

To run some testing, you can modify the timestamps in the file:

<stellent>/config/activestate.hda

Each scheduled system event will have an entry in this file. To test the logic on when your event will fire, you can set the timestamp to be one hour, one day, or even one year in the past. Then restart the server. You may need to reboot twice if you have two events that have the same 'eventCategory', such as this sample.

This component demonstrates:

  • Custom Resources
  • Filters with custom Java code


Installation

This component will be installed when you install the component bundle "HowToComponents.zip". If you have not done so already, use the Component Wizard or the Component Installer on the Admin Server, and install the bundle "HowToComponents.zip." Then enable this component, and restart the content server.

After installing the bundle, there will be the file "ScheduledEvent.zip" that can be used to install this component seperately.

If you do not wish to install the entire HowToComponents bundle, you may instead just install the "ScheduledEvents.zip" component and restart the server.
